說明

用於更新模型的權重值，降低模型預測結果與實際結果的誤差值

運作流程

假設有個模型如下：

  • 輸入值 i1 = 0.05 , i2 = 0.1
  • 輸入值通往第一層Hidden layer的權重值預設為:
    w1 = 0.15, w2 = 0.2 , w3 = 0.25 , w4 = 0.3
  • 第一層Hidden Layer(h1,h2)往output(o1,o2)的權重值預設為:
    w5 = 0.4 , w6 = 0.45 , w7 = 0.5 , w8 = 0.55
  • 第一層Hidden Layer Bias = 0.35 = b1
  • Output Layer Bias = 0.6 = b2
  • o1 實際答案:0.01 , o2 實際答案:0.99
    在進行反向傳遞之前，需要做一次Forward計算出Model Total Error valye

Forward flow

根據上面的圖示

  1. 算出hidden value
    h1的算法:
    $h1 = i1 * w1 + i2 * w2 + b1 = 0.050.15+0.10.2+0.35=0.3775$

  2. 標準化hidden value，以下稱作 out_h1 , out_h2
    使用activation function，在這個範例中，我們使用logistic function(sigmoid)
    將值規範在0~1之間
    sigmoid(h1) = out_h1 = 0.593269992
    sigmoid(h2) = out_h2 = 0.596884378

  3. 依前面的算法依此類推，根據out_h1 , out_h2算出out_o1 , out_o2
    out_o1,out_o2就是模型的預測結果:
    sigmoid(o1) = sigmoid(out_h1 * w5 + out_h2 * w6 + b2) = 0.75136507
    sigmoid(o2) = 0.772928465

  4. 算出模型的誤差值
    算誤差值的方法，可以使用取平均或是取平方差的方式計算
    在這個範例中，使用的是平方差
    => MSE

依照範例參數 => $E_total = ∑ 1/2(target_o1 - out_o1)^2$
因為我們有兩個輸出值，會有兩個誤差值，所以總誤差值為：
$E_total = E_o1 + E_o2 = 1/2(0.01 - 0.75136507)^2 + 1/2(0.99 - 0.772928465)^2$

算出誤差值後，就可以根據誤差值推算出每個權重值影響誤差值的程度，再透過學習率慢慢更新權重參數，降低模型預測的誤差值

Backward flow

Backward需要用到偏微分與chain role的觀念，可參考keyword連結


E對w5微分
意義：w5的權重值影響總誤差值的程度

根據chain role，可以將E對w5微分轉成下面的式子

  1. 算E對out_o1微分(out_o2依此類推)

  1. 算out_o1對net_o1微分(net_o1就是沒有經過sigmoid的原始值)

out_o2對net_o2微分算法同上

  1. 算net_o1對w5微分

  1. 透過自訂的learning rate =>符號唸eta
    更新w5權重值，假設我們設定rate=0.5

其他權重值的算法如同上面的流程一樣，比較要注意的是，例如:計算h1對誤差值的影響程度，根據模型flow會是兩個E_o1,E_o2相加所得

經過N次的權重更新後，模型的誤差值通常會慢慢地降低
